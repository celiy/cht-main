Components:
    - Loader: Barra com sliding infinito de loading.
    - Item: Mini card que tem mesmo estilo do select e radio em formato de card mas com ícone e cores diferentes.
    - OptionDrodown: Dentro do dropdown normal e select, uma opção que pode spawnar com click ou hover um dropdown do lado com mais opções e isso acontece infinitamente.
    - Avatar: Avatar de perfil, imagem redonda, opção para vários perfis com +[num] caso tenha mais que o desejado e status de online, away, etc.

    - FormHandler: Da handle de ações de form e etc. É um wrapper para os inputs dentro dele.
        - Quando usuário da submit ele envia todos os inputs pelo seus id's junto com os seus valores, quando envia para o back e o back retorna um erro com o nome do campo e mensagem, ele pega o nome do campo que é o id do input e retorna o erro para o front-end. No caso ele deve emitir por @errors um valor que contém os campos e erros de cada.
        - Passar a rota que é para enviar para o back;
        - Passar a ação: PUT, POST e etc;
        - Fazer automaticamente a ação de enviar os dados para o backend e outros afins relacionados com form;
        - @success;
        - @cancel;
        - @failure;
        - @errors;
        - Botão de submit e cancelar padrão. Usuário pode optar por usar botões padrões ou passar via slot. Se usar os botões padrões, o botão de cancelar padrão irá emitir @cancel.
        - automaticamente mandar forms do formato que aceita arquivos como midia;
        - Pressionar enter enquanto com um campo ativo no form da submit dele;
        - Opção de dar submit com Enter quando não há botões visiveis de submit dentro da tag do form;
        - Enter do textarea não afeta submit mas insere nova linha;
    
Custom components:
    - Select Search: Pesquisa local está pronta, apenas implementar pesquisa remota. Pesquisa remota muda o icone da lupa para o simbolo de loading.

    - Data Group: inclui o componente Data Table e Search Component, fazendo os dois trabalharem juntos; 
        - Data Table: Componente que recebe uma array de objetos, que é os dados a serem mostrados e uma outra array de objetos que é como será visualmente. Clicar em uma coluna da tabela ordena aquela tabela entre decrescente e crescente. Este componente inclui o componente Pagination e seleção de quantos items devem ser mostrados por página.

        - Search Component: Componente que é um input para pesquisa, botão de pesquisa, botão de reload e botão para selecionar filtros. Os filtros são passados como uma array de objetos e retorna uma array de strings que é os filtros usados.

        Data Group usa os filtros do Search Component e pode clicar nos headers do DataTable ordena por ordem alfabetica ou min para max juntamente com o filtro de pesquisa do Search.
